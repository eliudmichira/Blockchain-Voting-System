System Overview

The BBEMS is a secure, transparent electoral management system built on blockchain technology to ensure the integrity of Kenya's electoral processes. The system:

  • Authenticates voters via National ID and multi-factor authentication
  • Records votes as immutable transactions on a private Ethereum-based blockchain
  • Provides real-time monitoring and reporting capabilities
  • Allows for transparent tallying and result verification
  • Maintains voter privacy while ensuring verifiability

Administrator Roles

The system supports several administrative roles with different access levels:

Role Access Level Responsibilities
System Administrator Full system access System configuration, user management, security protocols
Election Administrator Election configuration Setting up elections, candidate management, constituency setup
Polling Station Administrator Station-level access Voter verification, local system management, support
Auditor Read-only access System monitoring, transaction verification, compliance checks
Technical Support Tiered access Issue resolution, system maintenance

System Setup

Hardware Requirements

  • Server Infrastructure:
    • Application Servers: Minimum 8-core CPU, 32GB RAM, 1TB SSD
    • Database Servers: Minimum 16-core CPU, 64GB RAM, 2TB SSD in RAID configuration
    • Blockchain Nodes: Minimum 8-core CPU, 16GB RAM, 500GB SSD
    • Load Balancers: Dual redundant configuration
  • Network Requirements:
    • Primary Fiber Connection: 100Mbps minimum dedicated line
    • Backup Connection: Secondary provider with automatic failover
    • VPN Infrastructure: For secure administrative access
  • Polling Station Equipment:
    • Verification Terminals: Core i5 or better, 8GB RAM, 256GB SSD
    • Backup Power: UPS with minimum 4-hour runtime
    • Network Equipment: 4G LTE backup connection

Software Installation

  1. Core System Installation: 
    # Clone the repository
git clone https://github.com/iebc/blockchain-voting-system.git

# Install dependencies
cd blockchain-voting-system
npm install

# Configure environment
cp .env.example .env
# Edit the .env file with your configuration

# Initialize the database
npm run init-db

# Deploy smart contracts to the blockchain network
npm run deploy-contracts
  2. Network Configuration:
    • Configure the private blockchain network
    • Set up validator nodes in multiple secure locations
    • Configure consensus parameters for optimal security and performance
    • Set up monitoring tools for network health
  3. Security Setup:
    • Install and configure firewalls
    • Set up intrusion detection/prevention systems
    • Configure SSL/TLS for all web interfaces
    • Implement DDoS protection measures

Election Setup

  1. Create a New Election:
    • Log in to the Admin Panel
    • Navigate to "Election Management" > "Create New Election"
    • Enter election details:
      • Name
      • Date
      • Type (Presidential, Parliamentary, County, etc.)
      • Applicable constituencies
    • Submit and review the created election
